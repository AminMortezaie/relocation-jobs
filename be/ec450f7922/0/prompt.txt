# KUCHUP — Design System Migration Spec
**For: implementation agent (Claude Code / dev)**
**Stack: Next.js + Tailwind CSS**
**Goal: migrate the entire application (marketing site + authenticated panel) from its current visual style to the "Night Flight" design system described below.**

---

## 0. Context for the agent

This is a full visual reskin of an existing product — a relocation/job-board SaaS ("Relocation Jobs", domain kuchup.com) — not a new build. Preserve all existing routes, data, and functionality. Only change: color tokens, typography, spacing/radius scale, and component visual treatment (cards, badges, buttons, nav). Do not change page structure, copy meaning, or business logic unless explicitly noted below.

Reference image: attached dark hero mockup ("KUCHUP — All your opportunities, in one place") — treat this as the north star for tone, not a pixel-exact template. Real product data (company names, roles, statuses) must replace any placeholder testimonial content shown in the mockup.

---

## 1. Design tokens

Create a single source of truth — `tailwind.config.js` theme extension + a `:root` CSS variables block — and reference tokens everywhere. Do not hardcode hex values in components.

### 1.1 Color

```css
:root {
  /* Base */
  --bg-base: #0A0E27;          /* page background */
  --bg-surface: #12173A;       /* card/panel background, ~90% opacity over base */
  --bg-surface-hover: #171C42;

  /* Borders */
  --border-subtle: rgba(255, 255, 255, 0.08);
  --border-hover: rgba(255, 255, 255, 0.16);

  /* Accents */
  --accent-primary: #FF6B35;       /* CTA buttons, primary actions, logo highlight */
  --accent-primary-hover: #FF7F52;
  --accent-blue: #3B82F6;          /* info badges, "role count" chips, icon accents */
  --accent-purple: #8B5CF6;        /* secondary status badges e.g. "Awaiting response" */
  --accent-green: #22C55E;         /* positive status e.g. "Visa-friendly", "CV ready" */

  /* Text */
  --text-primary: #F8FAFC;     /* headlines, high-emphasis */
  --text-secondary: #CBD5E1;   /* body copy, sub-headlines */
  --text-muted: #94A3B8;       /* captions, metadata, timestamps */
  --text-on-accent: #0A0E27;   /* text on orange buttons if using light-on-accent, else #FFFFFF */
}
```

**Gradient glows** (the depth effect behind cards/hero): radial-gradient blobs, 40–60% opacity, heavy blur (80–120px), placed behind card clusters and the hero illustration:
- Blue glow: `radial-gradient(circle, #3B82F6 0%, transparent 70%)`
- Purple glow: `radial-gradient(circle, #8B5CF6 0%, transparent 70%)`
- Orange glow: `radial-gradient(circle, #FF6B35 0%, transparent 70%)`

Use at most 2 glows visible in any single viewport — this is atmosphere, not confetti.

### 1.2 Typography

- **Display face** (headlines, hero text): a bold geometric/grotesk sans, tight tracking, weight 700–800. Recommended: **Manrope** or **Space Grotesk**.
- **Body face**: a humanist sans, weight 400–500, comfortable line-height (1.6). Recommended: **Inter**.
- **Utility face** (badges, metadata, timestamps, code): same as body, weight 500, smaller size, slightly wider letter-spacing for all-caps labels.

Type scale (rem, assuming 16px base):
| Role | Size | Weight | Line-height |
|---|---|---|---|
| Hero H1 | 3.5rem (mobile: 2.25rem) | 800 | 1.1 |
| Section H2 | 2rem (mobile: 1.5rem) | 700 | 1.2 |
| Card title | 1.25rem | 700 | 1.3 |
| Body | 1rem | 400 | 1.6 |
| Caption/badge | 0.75rem | 500 | 1.4, letter-spacing 0.02em |

### 1.3 Spacing & radius

- Base spacing unit: 4px, use Tailwind's default scale.
- Card radius: `rounded-2xl` (16px).
- Button radius: `rounded-lg` (8px).
- Badge/pill radius: `rounded-full`.
- Section vertical padding: 96px desktop / 56px mobile.

### 1.4 Elevation / glass effect

Cards use a translucent surface, not a flat fill:
```css
.card {
  background: var(--bg-surface);
  border: 1px solid var(--border-subtle);
  backdrop-filter: blur(12px);
  box-shadow: 0 8px 32px rgba(0,0,0,0.24);
}
.card:hover {
  border-color: var(--border-hover);
  transform: translateY(-2px);
  transition: all 200ms ease-out;
}
```

## 2. Core components

Build these as isolated, reusable components first. Every page in section 4 is composed from these — don't let any page invent a one-off variant.

### 2.1 Status badge / pill
Rounded-full, small padding (`px-3 py-1`), uppercase or sentence-case label, colored by status. Map existing product statuses to accent colors:

| Product status | Badge color |
|---|---|
| Looking to apply | `--accent-blue` |
| Applied | `--accent-green` |
| Awaiting response | `--accent-purple` |
| Rejected | `--text-muted` (desaturated, no bright accent — this is a "closed" state, don't make rejection visually loud) |
| Visa-friendly / CV ready | `--accent-green` |

### 2.2 Glass card
Base card per §1.4. Variants:
- **Feature card** (marketing): icon chip (bottom-left, circular, `--bg-surface-hover` fill), title, 1–2 line description, optional glow behind.
- **Company board card** (product): company name, country flag/tag, role count, list of open roles as sub-rows, status pill top-right, "N ago" timestamp.
- **Testimonial card**: avatar (circular, 48px), name + role, quote (1–2 sentences, body face), company logo row at bottom.

### 2.3 Buttons
- **Primary**: `--accent-primary` fill, `--text-on-accent` or white text, `rounded-lg`, medium weight, hover → `--accent-primary-hover` + slight scale (1.02).
- **Secondary/outline**: transparent fill, `1px solid var(--border-subtle)`, white text, hover → border becomes `--border-hover` + subtle bg tint.
- Never use more than one primary button per viewport section — it should be obvious what the one next action is.

### 2.4 Nav bar
Sticky, `--bg-base` at ~80% opacity with blur backdrop on scroll. Logo mark + wordmark left, links center/right, primary button ("Sign in" / "Get started") far right.

### 2.5 Search/filter bar
Rounded container (`rounded-xl`, `--bg-surface`), internal dividers as thin vertical `--border-subtle` lines between filter segments, primary button flush right. This is the "Job Description · Upload CV · Find Matches" bar pattern — reuse it anywhere the product needs a filter+action row (board filters, role search).

## 4. Page-by-page migration

Apply tokens/components in this order — each phase should be independently shippable and reviewable.

### Phase 1 — Landing page (`/`)
- Hero: H1 + subhead + primary/secondary button pair, bird illustration right (desktop), stacked below headline on mobile.
- "What you get" — 3 feature cards using real product concepts, not the mockup's placeholder ones:
  1. "Every relevant role, one place" — icon: search/globe
  2. "Track it like a pipeline" — icon: pipeline/kanban
  3. "Tailor your resume per job" — icon: document
- Live board preview section — pull 2–3 real company cards (e.g. from your existing catalog) rendered with the new card component and real status pills.
- Trust bar (free to use / no spam / privacy / ATS count) — thin single-row strip, icon + text pairs, muted.
- Footer — country links (**add Ireland here** — currently missing from the site's footer/filter list despite being a named target market), pricing, how-it-works, GitHub source/contributing.

### Phase 2 — Panel / board (authenticated app)
- Reskin the real board UI (company cards, role rows, status pills, filters) onto the same component set from §2 so the product doesn't feel visually disconnected from the marketing site post-login.
- Filter bar at top of board reuses §2.5 pattern.

### Phase 3 — Secondary marketing pages
- `/pricing`, `/how-it-works`
- Country landing pages: `/relocation-jobs-germany`, `/relocation-jobs-netherlands`, `/relocation-jobs-uk`, `/relocation-jobs-portugal`, and a new `/relocation-jobs-ireland` (create this page — Ireland is a current target market not yet represented in site content).

### Phase 4 — Cleanup
- Remove all old palette CSS/tokens once every page is migrated — no dead styles left behind.
- Contrast audit: `--text-muted` (#94A3B8) on `--bg-base` (#0A0E27) — verify ≥ 4.5:1 for body text use; if a caption use falls below, that's acceptable (WCAG AA large-text threshold), but primary body copy must pass.
- Mobile pass: hero illustration, 3-card grids, and the board's multi-column card layout all need defined stacking breakpoints (test at 375px and 768px).
- Reduced-motion: respect `prefers-reduced-motion` for hover transforms and any scroll-reveal animation.

---

## 5. Explicit non-goals

- Do not change routing, data fetching, or any business logic.
- Do not rewrite existing marketing copy wholesale — only adjust copy where the new component shape requires different length (e.g. shorter card descriptions).
- Do not invent new statuses or product features as part of this task — badge colors map to *existing* statuses only.
- Testimonials: use placeholder-but-honest content (e.g. "— from an early user") until real testimonials are collected; do not fabricate named people, companies, or quotes attributed to real individuals.
